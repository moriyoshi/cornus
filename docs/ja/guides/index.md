# ガイド

Cornus のあらゆる機能を対象にした、タスク指向の機能別手順集です。各ガイドは、実行に必要な背景だけを添えた、短くコピーして使えるレシピで構成されています。詳しい情報が必要な場合は、[CLI リファレンス](/ja/cli/)、[リファレンス](/ja/reference/deploy-spec)、[トピック](/ja/topics/remote-workflows)の **関連項目** を参照してください。

Cornus を初めて使いますか? まずは[クイックスタート](/ja/introduction/quick-start)を読み、その後ここへ戻って目的に合うレシピを探してください。複数の機能を組み合わせるエンドツーエンドのシナリオは、[クックブック](/ja/cookbook/)を参照してください。

## ガイドを探す

### [イメージをビルドする](/ja/guides/building-images)

Dockerfile をビルドして同梱レジストリへプッシュし、ビルド引数を渡し、キャッシュ / シークレット / SSH マウントを使い、名前付きビルドコンテキストを追加します。リモートサーバー上で遅延転送付きのビルドを実行し、リモートビルドキャッシュをインポート、エクスポートし、ルートレスでビルドする方法も扱います。

### [ワークロードをデプロイする](/ja/guides/deploying-workloads)

Compose プロジェクトまたは直接記述したデプロイスペックを Docker ホスト、ベアメタルの containerd ホスト、Kubernetes にデプロイします。削除、切り離し、スケール、ロールアウト、ワークロード内でのコマンド実行、クライアントローカルディレクトリのマウント、公開・非公開ポートへの接続も扱います。

### [Compose、devcontainers、docker CLI](/ja/guides/compose-devcontainers-docker)

Compose プロジェクトの起動・停止、サービスの確認と再ビルド、複数の Compose ファイル / 環境変数ファイル / プロファイルの使用、Dev Container の実行、Docker API プロキシ経由で Cornus サーバーを操作する通常の `docker` CLI の使い方を説明します。

### [リモートクラスターで作業する](/ja/guides/remote-clusters)

コマンドの接続先をリモートサーバーに設定し、接続プロファイルを作成し、クラスター内のサーバーへ自動でポート転送します。自身の Kubernetes へのアクセス権から短期間有効な資格情報を発行し、コンテキストを切り替え、サーバー経由で通信をルーティングする方法も扱います。

### [ネットワークのレシピ](/ja/guides/networking)

ローカルポートを転送し、SOCKS5 スプリットトンネルプロキシを実行し、conduit を選びます。hub オーバーレイ経由でワークロード同士を接続する方法も扱います。

### [トンネル](/ja/guides/tunnels)

ホスト型トンネルを介してワークロードポートを公開します。ngrok、SSH、Cloudflare、Tailscale の各バックエンドの段階的な設定手順を扱います。

### [イングレス](/ja/guides/ingress)

Kubernetes バックエンドで、ワークロードに公開 HTTP(S) ホスト名を割り当てます。自動導出または明示的なホスト名、cert-manager による TLS、パス / ポート / クラスに基づくルーティングを扱います。

### [エグレス](/ja/guides/egress)

VPN、企業プロキシ、外部ネットワークから隔離されたクラスターで利用するために、ルートルールまたは PAC ポリシースクリプトを用いて、リモートワークロードの外向きトラフィックを呼び出し元のネットワーク経由でルーティングします。

### [資格情報](/ja/guides/credentials)

LLM API キーを含む、呼び出し元が発行したシークレットを、イメージ、仕様、Pod 仕様に埋め込まずにワークロードへ仲介します。

### [レジストリとストレージ](/ja/guides/registry)

ファイルシステム、メモリ内、S3、GCS / Azure ストレージ上でレジストリを稼働させ、イメージをプッシュ、プルし、匿名プルを許可します。クラスターランタイムにレジストリを通知し、外部レジストリを使い、ガベージコレクションで領域を回収する方法も扱います。

### [サーバーを保護する](/ja/guides/security)

ベアラートークンを必須にし、JWT を発行し、JWKS による検証を行い、ID ごとの認可を伴う mTLS を有効化します。書き込みを保護しつつ匿名プルを許可する方法も扱います。

### [オブザーバビリティ](/ja/guides/observability)

OpenTelemetry のトレース、メトリクス、ログを有効にし、Prometheus `/metrics` エンドポイントを追加し、ログを設定し、liveness probe と readiness probe を構成します。

### [出力モード](/ja/guides/output-modes)

CLI の進捗と結果の表示形式を選びます。`auto`、`fancy`、`plain`、`json` (エージェントとスクリプト向けの NDJSON) と、色出力の制御を扱います。
